📌 Overview

MultiCam Director allows multiple phones to act as cameras while a laptop acts as a director. The system records:

  • 🎬 One final PROGRAM video (the main coach analysis film)
  • 📼 Separate recordings for each individual camera

This project is intentionally designed around real-world WebRTC limitations and avoids fragile approaches that break recording reliability.

---

✨ Features

  • 📱 Connect phones as cameras via QR code
  • 🎛 GO LIVE button for broadcast-style camera switching
  • 🎬 One continuous PROGRAM recording
  • 📼 Individual recordings per camera
  • 🔍 Camera controls: Zoom / Flip / Reset
  • 🏷 Camera renaming for clean exports
  • 🌐 Browser-only (no mobile app)
---

🧱 Tech Stack

  • WebRTC – real-time video transport
  • Socket.IO – signaling & control
  • HTML / CSS / JavaScript
  • Node.js

🧠 Why This Architecture

❌ The wrong approach

Switching video.srcObject during recording causes:

  • Blank frames
  • Corrupted recordings
  • Unexpected downloads
  • Browser-specific MediaRecorder bugs

✅ The correct approach (used here)


Phone Cameras (WebRTC)
        │
        ▼
Laptop Director
        │
   Canvas Compositor
        │
        ▼
   PROGRAM Recorder

All cameras are drawn into a single canvas. The canvas produces one stable MediaStream which is recorded continuously.

GO LIVE only changes what is drawn — not what is recorded. This mirrors real broadcast systems (OBS / vision mixers).

⚡ Quick Start


npm install
node server.js
  1. Open laptop.html
  2. Scan QR on phones → opens phone.html
  3. Use GO LIVE to switch cameras
  4. Start Recording
  5. Stop Recording → files download
---

🚀 Full Setup Rules (Important)

  1. Start the Node.js server first
  2. Open laptop.html
  3. Then connect phones

Opening phones before the laptop may cause missing peers.

⚠️ Network & Browser Limitations

🌐 Network

  • All devices must be on the same Wi-Fi router
  • ❌ Mobile hotspots are unreliable

🔐 HTTPS

Local IPs may be treated as insecure origins. HTTPS is recommended for production usage.

---

🚩 Chrome Flags (Local Development)

Android (Chrome)

  • Usually works without changes
  • If camera fails: open phone.html once and grant permissions

iPhone (Safari / Chrome)


chrome://flags
Insecure origins treated as secure
http://YOUR_LOCAL_IP:PORT

Only required for local testing. HTTPS removes this requirement.

---

📱 Mobile Device Tips

  • Keep phone screens ON
  • Lock orientation
  • Older phones may overheat or drop frames
  • If a camera drops, refresh only that phone
